How do I repair Win32/Sasser.worm? The system keeps on rebooting.

Hit
1472

System reboot is caused by Win32/Sasser.worm attacks. Win32/Sasser.worm spreads by exploiting MS04-011 vulnerability.

On infected systems, avserve.exe is created in Windows folder and 'number_up.exe' is created in Windows system folder. CPU usage increases up to 100% and the computer speed slows down after some time. C:\win.log file could get created as well. The unpatched system may shutdown automatically when it receives attacking packets. You must install MS04-011 vulnerability patches to prevent system shutdown caused by worm and odd packets using the same vulnerability.

 

AhnLab produces and distributes Win32/Sasser.worm removal tool to block packets that attack LSASS vulnerability (MS04-011) and cause system rebooting. If LSASS.EXE error occurs and Windows shuts down, please refer to the following steps. 1) Download V3 FirstBlock for Win32/Sasser.worm. Download (click) [V3 FirstBlock for Win32/Sasser.worm] 2) Run the downloaded v3sasser.exe file and click 'Start scan'. (Once it runs, packets are blocked. No other action is required.)

 

* If rebooting message appears before running V3 FirstBlock for Win32/Sasser.worm, unplug your LAN cable and run V3 FirstBlock for Win32/Sasser.worm after reboot, and then reconnect the LAN cable.While V3 FirstBlock for Win32/Sasser.worm is running, download MS04-011 patch for your OS and run it.Reboot your system.
